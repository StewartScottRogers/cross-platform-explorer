## Summary

Split from CPE-314. A full i18n pass — externalize every UI string into message catalogs, add a
locale store + formatting (dates/numbers), and a language setting. A large architectural change
with limited value for a single-user local desktop explorer, so parked at Low until there's
demand. The accessibility half of CPE-314 was completed separately.

## Acceptance Criteria
- [x] Locale store + message catalogs + a reactive `t()` translator with graceful fallback.
- [x] Locale-aware date/number formatting via `Intl`.
- [x] A language setting that switches the app live and persists the choice.
- [x] Ship more than one locale to prove the system (English + Spanish + German + French).
- [~] Externalize every UI string — the framework + primary chrome are migrated; the exhaustive
      per-component migration is the mechanical tail, split to [[CPE-481]].

## Resolution
Built the i18n **framework** and a working, visible language switcher; chose a zero-dependency
approach (Svelte stores) over a library, honoring PURPOSE.md's small-and-simple tiebreaker.

- **New `src/lib/i18n.ts`** — a `locale` writable (persisted to `localStorage`, restored on load,
  OS-language detection), 4 message catalogs (en/es/de/fr, English authoritative + the fallback), a
  reactive `t(key, params)` with `{name}` interpolation and locale→English→key fallback (a missing
  translation is never a blank label), and reactive `Intl` `formatDate`/`formatNumber`. **10 unit
  tests** (lookup, interpolation, fallback, live switch, orphan-key guard, Intl formatting,
  detection/persistence).
- **Language switcher** — a 🌐 Language menu in `MenuBar` switches the locale live with a check on the
  active one; the whole app re-renders reactively.
- **Primary chrome migrated** — `NavToolbar` (back/forward/up/refresh/search), `MenuBar` (titles),
  `Sidebar` (Agents/Repositories headers), `TabBar` (new/close). These prove the end-to-end path.

The exhaustive migration of the remaining components' strings is mechanical follow-through (safe to do
incrementally thanks to the English fallback) and is tracked as [[CPE-481]].

Verified: `npm run check` clean; **458 frontend tests pass** (incl. the 10 new i18n tests).
