        How the Agent Deck works
        
      

      

        
        

          
It all does one job: run a coding agent (Claude Code, aider, …) against a folder, using the provider and model you choose.


          
1 · Start an agent

          
Pick an Agent, a Provider, and a Model (leave Model blank for the agent's default), set the Working folder, then click Launch. Each Launch opens its own tab; the sidecar runs them all at once.


          
2 · Use your own API key — “Keys…”

          
Click Keys… to save a provider key. It's stored in your OS keychain, never in a file, and never shown again. Give it a label (e.g. “work”) to keep several keys per provider — a labelled key then appears in the Account picker. Check verifies a key with the provider before you save it.


          
3 · Save a reusable setup — “Save setup”

          
A setup remembers your provider + model choice under a name. It does not save your key — keys live in Keys…. Type a name, click Save setup, then pick it later from the Setup dropdown. Setups are per-agent.


          
4 · Keep agents current — “Manage agents ▾”

          
Check for updates now fetches new/updated agents (signed); Auto-update on open does it automatically; Pin this agent freezes one agent's version; Reset to the shipped agents undoes updates.


          
5 · Recent sessions — “Recent…”

          
Lists past sessions (kept across restarts). Relaunch reopens one with the same agent/provider/model/folder; or view its (secret-redacted) transcript.


          
Permissions

          
The console runs as its own sandboxed process and only gets what you allow. Secrets is the permission that lets it store your API keys — if you skip it, saving keys won't work and you'll simply be asked again next time you open the console. Review or revoke permissions in the app's sidecar settings.

        


        

          
See several agents at once

          
With sessions open, click ▦ Grid to tile every agent side by side (they auto-arrange);
            click ▤ Tabs to go back to one at a time. In grid view, click a tile to focus it, or
            press Ctrl+Alt+Arrow to move between tiles. Each tile shows the agent's colour chip,
            name, and usage. Nothing about the single-tab view changes.

        


        

          
What a swarm is

          
A swarm is several agents working the same folder together: a coordinator plans
            and dispatches, and one builder per task line does the work (experimental).

          
Running one

          
Click Run swarm ▾, enter one task per line, then Start. The swarm panel lists
            each member and its status; Hide tucks it away while the swarm keeps running.

          
New to swarms?

          
Pick a demo from the dropdown and click Load demo — it fills in a ready-made,
            safe example (it only creates files, nothing is deleted). Choose your agent, press Start,
            and watch a swarm work end-to-end without writing any tasks yourself.
